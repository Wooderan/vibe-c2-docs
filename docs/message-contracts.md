# Message Contracts

This page defines initial RabbitMQ message contract shapes.

## `inbound.agent_message`

Purpose: carry routing metadata plus an opaque encrypted implant payload from channel modules to core C2.

### Contract principles

- `message_id` is generated by channel module (first trusted backend hop).
- Implant payload is opaque to channel modules.
- Only core C2 services can decrypt/verify payloads.
- Core resolves keys using routing identifiers (for example `implant_id`, `session_id`).

### Envelope (v1)

```json
{
  "message_id": "01JNX6R8VQ2H3CN4K9EJ1T2Z7M",
  "correlation_id": "sess-2b77df",
  "type": "inbound.agent_message",
  "version": "1.0",
  "timestamp": "2026-03-09T21:05:12.481Z",
  "source": {
    "module": "channel-http",
    "module_instance": "channel-http-1",
    "transport": "http",
    "tenant": "default"
  },
  "routing": {
    "implant_id": "a-9f7c2d19",
    "session_id": "s-2b77df",
    "listener_id": "listener-main"
  },
  "encrypted": {
    "scheme": "x25519-chacha20poly1305-v1",
    "blob_encoding": "base64",
    "ciphertext": "QkM4V1R...",
    "nonce": "N3h3b...",
    "tag": "i8Yg...",
    "aad": {
      "type": "inbound.agent_message",
      "version": "1.0",
      "implant_id": "a-9f7c2d19",
      "session_id": "s-2b77df"
    },
    "key_hint": "kid:imp-a-9f7c2d19:v3"
  },
  "meta": {
    "receive_count": 1,
    "trace_id": "tr-6fd92d8b"
  }
}
```

### Required top-level fields

- `message_id`: unique backend-generated message identifier (`ULID` recommended).
- `correlation_id`: end-to-end flow/session identifier.
- `type`: fixed value `inbound.agent_message`.
- `version`: schema version string (start with `1.0`).
- `timestamp`: RFC3339 UTC timestamp.
- `source`: producing channel-module metadata.
- `routing`: identifiers used for routing and key lookup.
- `encrypted`: encrypted payload container.

### Required fields (v1)

- `routing.implant_id`
- `routing.session_id`
- `encrypted.scheme`
- `encrypted.blob_encoding`
- `encrypted.ciphertext`

### Validation rules (v1)

- Reject when `type != inbound.agent_message`.
- Reject when required fields are missing.
- Reject when `timestamp` is not parseable RFC3339.
- Reject when `blob_encoding` or `scheme` is unsupported.
- Treat payload as opaque before core decrypt.
- Accept unknown extra fields for forward compatibility.

### RabbitMQ routing (proposal)

- Exchange: `c2.inbound`
- Routing key: `agent.message.<transport>`
- Queue (core consumer): `core.inbound.agent_message.v1`
- DLQ: `core.inbound.agent_message.v1.dlq`

### Processing expectations

- Channel module publishes at-least-once.
- Core server treats `message_id` as idempotency key.
- Core resolves keys from routing identifiers and decrypts/verifies message.
- Translator/provider processing starts only after successful core decrypt.
